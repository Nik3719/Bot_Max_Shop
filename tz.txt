Интернет-магазин на базе MAX-бота
с синхронизацией Google Sheets ↔ SQLite
1. Цель и общее описание
Разработать MAX-бота (платформа MAX) с функционалом интернет-магазина. Товары ведутся в Google Sheets администратором и синхронизируются в SQLite-базу данных бота. Покупатели проходят регистрацию, просматривают ленту товаров с фотографиями и оформляют заявки на покупку. Все заявки немедленно рассылаются администраторам с полной информацией о товаре и покупателе.

2.3. Настройка доступа к Google Sheets API
Для работы с API необходим Service Account Google:
Создать проект в Google Cloud Console → включить Google Sheets API и Google Drive API
Создать Service Account → скачать JSON-ключ → сохранить как credentials.json в корень проекта
Открыть таблицу → поделиться с email сервисного аккаунта (роль: Читатель для загрузки, Редактор если нужна обратная запись)
ID таблицы взять из URL: https://docs.google.com/spreadsheets/d/{SPREADSHEET_ID}/edit
🔐  Файл credentials.json добавить в .gitignore — он содержит секретный ключ и не должен попасть в репозиторий.

4. Синхронизация Google Sheets ↔ SQLite
4.1. Направление 1: Google Sheets → SQLite (основное)
Запускается администратором командой /sync_from_sheets. Алгоритм:
Прочитать все строки листа «Товары» через Google Sheets API (pygsheets или gspread)
Валидировать каждую строку: проверить обязательные поля (product_id, name, price), пропустить строки с ошибками (логировать)
Для каждой валидной строки выполнить UPSERT в таблицу products:    • Если product_id уже есть → UPDATE всех полей    • Если product_id новый → INSERT
Товары, которые есть в БД (is_active=1), но отсутствуют в текущем листе → установить is_active = 0 (мягкое удаление)
Записать результат в sync_log
Отправить администратору отчёт: добавлено X, обновлено Y, скрыто Z
4.2. Направление 2: SQLite → Google Sheets (обратная запись)
Запускается администратором командой /sync_to_sheets. Используется для публикации заявок или обновления статусов в таблице. Алгоритм:
Прочитать все заявки (orders) со статусом 'new' из SQLite
Записать их на второй лист «Заявки» в Google Sheets (append строк)
Обновить статус orders → 'viewed' для перенесённых заявок
Записать результат в sync_log
ℹ️  Лист «Заявки» в Google Sheets — только для информирования администратора. Управление статусами заявок происходит через бота, не через таблицу.

4.4. Автосинхронизация (опционально)
Дополнительная возможность: настроить автосинхронизацию по расписанию через APScheduler:
# main.py — опциональный планировщик
from apscheduler.schedulers.asyncio import AsyncIOScheduler
 
scheduler = AsyncIOScheduler()
scheduler.add_job(auto_sync_job, 'interval', hours=SYNC_INTERVAL_HOURS)
scheduler.start()
 
async def auto_sync_job():
    result = sync_sheets_to_db(admin_user_id=None)  # None = автоматически
    # Уведомить всех администраторов о результате
    for admin_id in ADMIN_IDS:
        await bot.send_message(admin_id, f'🔄 Автосинхронизация: +{result["inserted"]} /{result["updated"]} скрыто {result["deactivated"]}')

7. Оформление заявки на покупку
7.1. Процесс оформления
Покупатель нажимает «🛒 Оформить заявку» под товаром
Бот показывает карточку товара + данные покупателя и спрашивает подтверждение
Опционально: бот предлагает ввести комментарий («Укажите удобное время звонка или любой комментарий. Нажмите /skip чтобы пропустить.»)
Покупатель подтверждает → INSERT в orders → уведомления всем администраторам

8. Функции администратора
8.1. Определение администратора
Администратор определяется по MAX ID в файле конфигурации. Никакой базы данных или регистрации для администраторов не требуется:
# config.py
ADMIN_IDS = [123456789, 987654321]  # MAX ID администраторов
# Декоратор проверки
def admin_only(func):
    async def wrapper(message, *args, **kwargs):
        if message.from_user.id not in ADMIN_IDS:
            await message.answer('⛔ Команда доступна только администраторам.')
            return
        return await func(message, *args, **kwargs)
    return wrapper
